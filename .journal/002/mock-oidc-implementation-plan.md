## Global conventions & invariants

These hold across every slice and are non-negotiable (TDD §1, §5; Decisions D-1..D-5).

- [ ] **Identity constants (verbatim, TDD §1):** module `github.com/meigma/mock-oidc`; binary + entrypoint dir `cmd/mock-oidc` (`cmd/mock-oidc/main.go`); OpenAPI `apiTitle` = `mock-oidc`; OTel `serviceName` = `mock-oidc`; GHCR image `ghcr.io/meigma/mock-oidc`.
- [ ] **Env config (TDD §1):** Viper prefix `MOCK_OIDC_`; **plus** unprefixed upstream-parity aliases via explicit `viper.BindEnv`: `SERVER_HOSTNAME`, `SERVER_PORT`, `PORT`, `JSON_CONFIG`, `JSON_CONFIG_PATH`, `LOG_LEVEL`. Precedence: `SERVER_PORT` > `PORT` > `8080`; JSON source `JSON_CONFIG` (inline) > `JSON_CONFIG_PATH` (file) > `./config.json` > defaults. `LOGBACK_CONFIG` accepted as a **no-op**.
- [ ] **Hexagon dependency rule (TDD §5):** three inward tiers — **core** `internal/oidc` (root files only), **adapters** `internal/oidc/{httpapi,controlapi,signing,memory}` + `internal/adapter/http`, **edges** `internal/app`/`internal/cli`/`internal/config`. Adapters never import each other; they meet only at the composition root. The core reaches crypto/HTTP/IO/time **only** through ports in `ports.go`/`clock.go`.
- [ ] **depguard crypto ban is narrowed (TDD §5c, Decision D-1):** the `oidc-core` rule denies only **key-bearing signing/JOSE** packages (`crypto/rsa`, `crypto/ecdsa`, `crypto/ed25519`, `crypto/tls`, `crypto/x509`, go-jose) + transport/framework/DB packages — **not** blanket `crypto`. The keyless PKCE S256 transform (`crypto/sha256`, `crypto/subtle`, `encoding/base64`) and `encoding/json` stay in the core. `doc.go`, the depguard rule, and `arch_test.go` must state this carve-out **word-for-word identically**.
- [ ] **Parse-don't-validate at the edge (TDD §5):** no `map[string]any`, bare `string` grant types, or raw `url.Values` cross into `internal/oidc`. Inbound form/query/JSON/header bytes become typed commands at the adapter; the outbound capture edge feeds `oidc.NewCapturedRequest` only stdlib primitives so the core never names `net/http`/`net/url`.
- [ ] **Routing:** single-segment `/{issuer}/…` path-param routing (`IssuerID` rejects `/`); the reserved `/_mock/` prefix hosts the container-first control plane; nested/multi-segment issuers are a **named parity gap** (Decision D-2).
- [ ] **The five resolved decisions (TDD Decisions, normative):**
  - **D-1** — domain-purity crypto carve-out confirmed (above).
  - **D-2** — multi-segment issuer IDs are an accepted, documented parity gap; `IssuerID` stays single-segment.
  - **D-3** — CORS **default ON, reflect-origin** with `Access-Control-Allow-Credentials: true` (preflight `Allow-Methods POST,GET,OPTIONS`, echo requested headers, `204`); `CORSAllowedOrigins` tightens to an allowlist. Applied in Slice 1 transport setup, hardened in Slice 6.
  - **D-4** — self-verification accepts both `typ=JWT` **and** `at+jwt` (RFC 9068) in `TokenVerifier`.
  - **D-5** — strip Huma's `SchemaLinkTransformer` in `NewAPI` (clear `cfg.Transformers` + the link `OnAddOperation` hook) so protocol JSON is clean and the fixed discovery field order holds; `Body []byte` fallback if any residual transform interferes. Applied in Slice 1.

## Cross-cutting: Testing Strategy & Quality Gates

This section is the load-bearing contract every slice's DoD plugs into. It defines the four-rung R1–R4 test pyramid (TDD §8.3, §8.11), the concrete files/packages each rung lives in, and a copy-paste **per-slice acceptance-gate checklist** the roadmap slices reference instead of re-specifying. **Doctrine: functional-first.** Per TECH_NOTES and TDD §8.3, no feature is "complete" until its functional (R2) and container (R3) rungs are green — a passing R1 unit suite alone never closes a slice. Unit tests protect the *design*; functional/container tests protect the *promise* (an unmodified OIDC client works against the shipped artifact).

**Which image R3 boots (read once, applies everywhere).** In Slices **0–5** the container under test is the **locally-built `mock-oidc:dev`** image — produced by `mise run image-local` (melange→apko→`docker load`) and referenced by that local tag from `testcontainers-go`. The published `ghcr.io/meigma/mock-oidc` tag does **not** exist until Slice 6's release chain runs (TDD §8.10 R4), so R3 tests in Slices 0–5 that name `ghcr.io/...` would fail to pull. The design's C.11 worked example hardcodes `ghcr.io/meigma/mock-oidc:latest` (TDD C.11, line 3303); the R3 helper reads the tag from an env var (`MOCK_OIDC_IMAGE`, default `mock-oidc:dev`) so the same suite runs against the local build pre-Slice-6 and the published artifact in Slice 6. **Reserve `ghcr.io/meigma/mock-oidc` for the Slice 6 published-artifact verification only.**

### The pyramid (what each altitude is for)

| Rung | Altitude | Where it lives | Command | Proves |
|---|---|---|---|---|
| **R1 Unit** | typed core `internal/oidc` | `*_test.go` beside each domain file; ports faked by `internal/oidc/mocks` | `moon run test` (`go test ./...`) | Constructors reject bad input; services orchestrate ports; invariants hold — no IO, no crypto, no `httptest`. |
| **R1-arch** | import gate | `internal/oidc/arch_test.go` | `moon run test` | Core import graph is transport/signing-free (the layering rule, §5/§10). |
| **R2 Functional** | wired adapter | `internal/oidc/httpapi/*_test.go`, `internal/oidc/controlapi/*_test.go` | `moon run test` | HTTP contract (status/headers/JSON/JWT bytes) correct in-process over **real** `signing`+`memory` adapters. |
| **R3 Container** | shipped image | `internal/integration/*_test.go` (`//go:build integration`) | `mise run image-local` then `moon run test-integration` | The image behaves for off-the-shelf clients; zero-config boot; real JWKS verification. **Boots `mock-oidc:dev` in S0–S5; `ghcr.io/meigma/mock-oidc` in S6.** |
